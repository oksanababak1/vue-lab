2. ПОМИЛКА TYPESCRIPT (ТЕСТУВАННЯ ТИПІЗАЦІЇ):

При спробі передати число замість булевого значення у компоненті UserCard:

Код з помилкою:
<UserCard username="Student_3" :isActive="123" />

Отримана помилка:
Type 'number' is not assignable to type 'boolean'. ts-plugin(2322)

Детальне пояснення 
UserCard.vue(5, 3): The expected type comes from property 'isActive' 
which is declared here on type '{ readonly username: string; readonly 
isActive: boolean; }'
Компонент UserCard у файлі UserCard.vue має типізований пропс isActive,
який очікує булеве значення (true або false). Коли я спробувала передати 
число 123 замість булевого значення, TypeScript одразу виявив помилку 
на етапі написання коду.

VS Code підкреслив значення "123" червоною лінією, а в терміналі з'явилося
повідомлення про помилку типізації. Це не дозволило скомпілювати застосунок
з неправильними типами даних.

Це демонструє головну перевагу статичної типізації - помилки виявляються 
ДО запуску програми, а не під час її роботи, коли користувач вже може 
зіткнутися з несправностями.
3.2. Переваги Vue Router:
   - Дозволяє створювати односторінкові застосунки (SPA)
   - Навігація між сторінками без перезавантаження всієї сторінки
   - Підтримка ліниво завантаження (lazy loading) для оптимізації
   - Зручна навігація через компонент RouterLink
   - Можливість передавати параметри через URL
Vue Router робить навігацію швидкою - при переході між сторінками 
   браузер не перезавантажує всю сторінку. Це покращує користувацький досвід.
   
   Ліниве завантаження означає що код для сторінки завантажується тільки 
   коли користувач на неї переходить, що робить перше завантаження швидшим.
3.3. Організація проєкту:
   - Чітка структура папок (views для сторінок, components для компонентів)
   - Розділення відповідальності (кожен компонент має свою задачу)
   - Можливість перевикористання коду через компоненти
   - Composition API (<script setup>) спрощує написання логіки
Проєкт має зрозумілу структуру - сторінки в папці views, компоненти 
   в components, маршрути в router. Це робить код організованим.
   
   Компоненти можна використовувати багато разів - наприклад, UserCard 
   я використала двічі на одній сторінці з різними даними.
3.4. Практичні навички, які я отримала:
   - Навчилася створювати Vue 3 проєкт з нуля за допомогою CLI
   - Розумію як працює маршрутизація в односторінкових застосунках
   - Вмію створювати типізовані компоненти з defineProps<{}>
   - Знаю як виправляти типові помилки конфігурації (шляхи, кеш)
   - Розумію різницю між звичайним JavaScript та TypeScript
